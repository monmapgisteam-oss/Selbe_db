# Сэлбэ — Suitability Modeler

Сэлбэ дэд төвийн 52 бүсийг **хот төлөвлөлтийн үзүүлэлт** ба **санхүүгийн үр ашиг**-аар
үнэлж, улаан→ногоон өнгөөр дүрслэх ArcGIS Maps SDK for JavaScript вэб аппликейшн.

## Ажиллуулах

```bash
npx http-server . -p 5178 -c-1
# → http://localhost:5178
```

Барилтын алхам шаардлагагүй — ArcGIS SDK 4.33 CDN-ээс ES module хэлбэрээр ачаална.

## Бүтэц

| Файл | Үүрэг |
|---|---|
| `js/config.js` | Үйлчилгээний URL, БНбД босго утга, жин, нэгж үнэ |
| `js/data.js` | Давхарга татах, бүсээр нэгтгэх, орон зайн тооцоо |
| `js/score.js` | Оноолт, жигнэсэн нийлбэр, өнгөний градиент |
| `js/app.js` | Газрын зураг, UI, дүрслэл |

## Загварын логик

**Ерөнхий оноо = Хот төлөвлөлт × 50% + Санхүү × 50%** (дээд талын гулсуураар өөрчилнө)

### Хот төлөвлөлтийн 8 үзүүлэлт (анхны жин)

| Үзүүлэлт | Жин | Эх сурвалж |
|---|---|---|
| **Хүн амын нягтшил (хүн/га)** | **24%** | Оршин суугч ÷ **полигоны талбай** (Shape__Area) |
| **FAR** | **19%** | Бүсийн **`FAR_HUVI` ÷ 100** |
| Нэг хүнд ногдох ногоон байгууламж | 16% | Ногоон байгууламж ÷ оршин суугч |
| Зогсоолын хангамж | 10% | `ET_NIIT` ÷ зогсоолын хэрэгцээ |
| BCR | 8% | Бүсийн **`BCR` × 100** |
| Нийтийн тээврийн хүртээмж | 8% | Автобус / LRT-BRT зогсоол хүртэлх зай |
| Алхалтын бүсийн хамрах хүрээ | 8% | Цэцэрлэгт хүрээлэнгийн алхалтын полигон ∩ бүс |
| Нийгмийн дэд бүтцийн хүртээмж | 8% | Сургууль, цэцэрлэг, эмнэлэг, цагдаа — оршин суугчид ногдох хүртээмж (2SFCA) |
| Инженерийн дэд бүтцийн хүртээмж | 7% | Цэвэр ус / бохир / дулааны шугам хүртэлх зай |

**Нийгмийн дэд бүтцийн хүртээмж** — байгууламжийг барилгын `Зориулалт_m`-оос
ялгаж, хүчин чадлыг `Total_population` талбараас авна.
*Цэцэрлэг (300 м, 50 суудал/1000 хүн)* ба *сургууль (500 м, 110/1000)* нь **2SFCA**
аргаар: байгууламжийн суудлыг үйлчлэх радиус доторх **оршин суугчдын тоонд**
хуваарилж, бүсэд ногдох суудлыг нормтой харьцуулна (нэг сургууль олон бүсэд
давхар тоологдохгүй). *Эмнэлэг (1000 м)*, *цагдаа (1500 м)* нь зайгаар — радиус
дотор 100%, 2×радиусаас цааш 0%. Жин: цэцэрлэг 30, сургууль 30, эмнэлэг 25,
цагдаа 15. Оршин суугчгүй бүсэд утга гарахгүй (өгөгдөлгүй).
Тохиргоо: `config.js` → `SOCIAL_FACILITIES`.

Хүн амтай холбоотой үзүүлэлт (нягтшил, FAR) нь хот төлөвлөлтөд хамгийн чухал
тул нийт жингийн **43%**-ийг эзэлнэ.

### Оноолт — нормын хатуу горим

- **Норм хангасан** → **100 оноо** (ногоон)
- **Норм зөрчсөн** → **дээд тал нь 44 оноо**, хатуу хязгаар руу 0 хүртэл буурна

44 нь «Дунд»-ын босго (45)-аас доогуур тул нормоос гарсан утга **шууд улаан/улбар
шар** болж харагдана. Жишээ: нягтшил 499 хүн/га (норм 450) → **35 оноо, улаан**
(хуучин логикоор 80 оноо, ногоон байсан).

[config.js](js/config.js) → `STRICT_NORM = false` болговол хуучин зөөлөн
шугаман буурах горимд шилжинэ. Бүх босго утгыг баруун талбараас засварлана.

### Зогсоол

**Хангамж** нь бүсийн давхаргын `ET_NIIT` (= `ET_IL` ил + `ET_DALD` далд —
52/52 бүст тэнцүү нь шалгагдсан). Нийт **5,759** зогсоол (3,984 ил + 1,775 далд).

**Хэрэгцээг** баруун талбарын «Зогсоолын хэрэгцээ» хэсгээс 3 аргаар тооцуулж
харьцуулна:

| Арга | Нийт хэрэгцээ | Хангалт | Дутагдалтай бүс |
|---|---|---|---|
| `NORM_ZOGS` эх өгөгдөл | 27,706 | 21% | 36 |
| Өрхийн тоо × 1.0 | 11,709 | 49% | 16 |
| Хүн ам × 300/1000 | 14,456 | 40% | 26 |

Коэффициент нь гулсууртай (өрхөд 0.2–2.0, 1000 хүнд 50–600).
Бүс сонгоход ил/далд задаргаа, зөрүү, хангалтын % харагдана; лавлагаанд
`NORM_ZOGS`, `SELBE_NIIT`, барилгын `Parking` нийлбэрийг хамт харуулна.

### Эдийн засгийн шинжилгээ («Эдийн засаг» таб)

**1. Дэд бүтцийн өртөг — үйлчилгээ тус бүрээр.** Нэгж үнийн талбартай **24**
давхаргыг геометрийн төрлөөр нь өөр өөрөөр нийт өртөг болгоно
([config.js](js/config.js) → `COST_LAYERS`):

| Геометр | Нэгж үнэ гэж юу вэ | Тооцоо |
|---|---|---|
| Цэг (5) | нэг байгууламжийн **бүтэн** өртөг | Σ үнэ |
| Шугам (16) | **100 м** бүтээн байгуулалтын өртөг | Σ үнэ × `urt_m` ÷ 100 |
| Талбай (3) | **1 м²**-ийн өртөг | Σ үнэ × `talbai_m2` |

⚠️ **Үнийн талбарын нэр давхарга бүрд ижил биш** — тохиргооноос `priceField`,
`qtyField`, `divisor`-оор дарж бичнэ:

| Давхарга | Талбар | Онцлог |
|---|---|---|
| Ихэнх (21) | `negj_une` | ердийн |
| Авто зам (5) | `negjune_km` + `urt_km` | үнэ нь **₮/км** → `divisor: 1` |
| Гүүрэн байгууламж (12) | `niit_une_sh` | шугаман геометртэй ч үнэ нь **ширхэгт** → `kind: point` |
| Инженерийн бэлтгэл (15) | `negj_une_100m` | нэгж үнэ объект бүрд **хувьсана** |

Үнэгүй (өртөгт ороогүй) давхарга: Автобусны чиглэл (6), Дулаан дамжуулах
хуваарилах төв (7), Зам-полигон (29 — авто замыг шугамаар нь тооцсон тул давхардахгүй).

**2. 1 га-д ногдох төсөв** = нийт өртөг ÷ **156 га** (`PROJECT_AREA_HA`)

**3. Бүсийн зардал** = 1 га-гийн төсөв × бүсийн `Area` (га).

> Тоон дүн нь ArcGIS дээрх нэгж үнээс шууд хамаарах тул эх өгөгдөл засагдахад
> өөрчлөгдөнө (жишээ: 2026-07-21-нд нийт өртөг 886.9 → 897.9 тэрбум ₮ болсон).

**4. Ашиг / алдагдал**
```
Орлого = Σ negj_une (4.7 сая ₮/м²) × Барилгын_нийт_талбай_m2
         ↑ зөвхөн Barilga_ty ≠ "Одоо байгаа" барилга
Ашиг   = Орлого − Зардал
Зардлын эзлэх хувь = Зардал ÷ Орлого × 100
```

**«Одоо байгаа» барилга орлогод ОРОХГҮЙ** — аль хэдийн борлуулагдсан/ашиглалтад
орсон тул төслийн ирээдүйн орлого биш ([data.js](js/data.js) → `isSellable`).

| Barilga_ty | Барилга | Талбай | Орлогод |
|---|---|---|---|
| Төлөвлөсөн | 231 | 1,390,206 м² | ✓ |
| Баригдаж байгаа | 113 | 666,240 м² | ✓ |
| **Одоо байгаа** | **24** | **7,204 м²** | **✗ (−33.9 тэрбум ₮)** |

> Энэ шүүлт нь **зөвхөн орлогод** үйлчилнэ. Хүн ам, нягтшил, FAR, зогсоол зэрэг
> хот төлөвлөлтийн үзүүлэлт бүх 368 барилгыг хэвээр тооцно.

| | Дүн (2026-07-21) |
|---|---|
| Дэд бүтцийн нийт өртөг | 897.9 тэрбум ₮ |
| 1 га-д ногдох төсөв | 5.76 тэрбум ₮/га |
| Бүсүүдэд ногдох зардал | 689 тэрбум ₮ |
| Борлуулалтын үнэлгээ (нийт барилга) | 9,699.2 тэрбум ₮ |
| Үүнээс орон сууц | 4,809.8 тэрбум ₮ |
| **Ашиг** | **≈9,010 тэрбум ₮** |
| Зардлын эзлэх хувь | ≈7.1% |
| Ашигтай бүс | 41 / 52 |

Оноолт (`ECON_SCORE`): зардлын эзлэх хувь **≤20% = 100 оноо**, **100% (ашиг тэг)
= 0 оноо**. Орлогогүй мөртлөө зардалтай 9 бүс нь «өгөгдөлгүй» биш **цэвэр
алдагдал** тул 0 оноотой улаанаар харагдана.

### Худалдаалагдах барилгын шүүлтүүр

Санхүүд **зөвхөн төлөвлөж буй болон баригдаж буй** барилга орно. Зориулалтаар
үйлчилгээ (бүхий л төрлийн), орон сууц, үзвэр, хүнсний зах, үйлчилгээтэй орон
сууц, оффис үйлчилгээ, single/town house орно.

| | Барилга | Талбай |
|---|---|---|
| **Худалдаалагдах** | **199** | **1,163,165 м² (76.3%)** |
| Хасагдсан | 169 | 360,669 м² |

Хасагдсан томоохон зүйлс: Хүүхэд/ахмадын хөгжлийн төв 134,400 м², ХТП 86,644 м²,
авто зогсоол 52,878 м², сургууль 35,749 м², дулааны станц 12,040 м², цэцэрлэг
10,469 м², спорт комплекс 8,400 м², эмнэлэг 6,272 м².

> Шүүлтүүр нь **зөвхөн санхүүд** үйлчилнэ. FAR, BCR, хүн амын нягтшил зэрэг хот
> төлөвлөлтийн үзүүлэлт нь **бүх 368 барилгыг** хэвээр тооцно.

Баруун талбарын «Худалдаалагдах барилгын шүүлтүүр» хэсгээс төлөв, зориулалт
бүрийг checkbox-оор асаах/унтраах боломжтой (тус бүрийн м² ба тоо хамт харагдана).

### Худалдаалах өртгийн суурь — select by location

Барилгыг бүсэд оноох аргыг санхүүгийн хэсгээс сонгоно:

| Арга | Бүсэд оногдсон барилга | Нийт ашигтай талбай |
|---|---|---|
| **Байршлаар** (select by location) — *анхны* | **368** | **1,523,834 м²** ✓ |
| `ZONE_ID` талбараар | 349 | 1,452,786 м² |

*(Дээрх нь шүүлтүүрээс өмнөх нийт дүн — шүүлтүүрийн дараа 1,163,165 м² худалдаалагдана.)*

**Байршлаар** нь барилгын **төв цэг** аль бүсийн дотор байгаагаар онооно
(`geometryEngine.contains`). Төв цэгээр авснаар барилга яг нэг бүсэд л ороод,
зааг дээрх барилга давхар тоологдохгүй — тиймээс бүсүүдийн нийлбэр эх
өгөгдөлтэй **яг тэнцэнэ**. Төв цэг ямар ч бүсэд орохгүй бол хамгийн ойрын бүс
рүү `SNAP_METERS = 100` м хүртэл наана ([data.js](js/data.js)).

`ZONE_ID` талбараар бодвол **19 барилгын утга хоосон** учир 71,048 м² унаж,
худалдаалах өртөг **334 тэрбум ₮-өөр дутуу** гарна. Мөн 6 барилгын `ZONE_ID`
нь бодит байршилтайгаа зөрдөг.

Санхүүгийн хэсэгт нэгтгэсэн талбай эх өгөгдлийн нийлбэртэй таарч байгаа эсэхийг
байнга шалгаж харуулна («бүрэн таарсан ✓» / хэдэн м² унасныг улаанаар).

> **Анхаар:** үйлчилгээний давхарга бүр 2,000 объектын хязгаартай тул
> [data.js](js/data.js)-ийн `fetchAll` нь **хуудаслаж** татдаг. Үгүй бол
> 3,200 объекттой «Гадна дулаан», 2,220 объекттой «Явган хүний зам» дутуу ирж
> өртөг 582 биш 534 тэрбум ₮ гэж багаар гарч байсан.

## UI боломж

- **Тайлбар — шүүлтүүр**: гистограммын багана дээр дарж тухайн оноотой бүсүүдээр
  шүүнэ. Газрын зураг дээр бусад бүс бүдгэрч, эрэмбэ мөн шүүгдэнэ. Дахин дарж цуцална.
- **Хураах товч**: Хот төлөвлөлтийн жин, Санхүүгийн дүн шинжилгээ, Тайлбар,
  Дэлгэрэнгүй, Давхарга, Ногоон ангилал — гарчиг дээр дарж хураана.
- **Горим тус бүрийн задаргаа**: дээд табыг сольход зүүн талын "Дэлгэрэнгүй" карт
  тухайн горимын гүнзгий үзүүлэлтийг харуулна:
  - *Ерөнхий* — хот/санхүүгийн дундаж, ангиллын тоо, хоёр талын хамгийн их зөрүүтэй бүс
  - *Хот төлөвлөлт* — үзүүлэлт бүрийн дундаж оноо, жин, нормоос доогуур бүсийн тоо
    (мөр дээр дарвал тухайн үзүүлэлт рүү шилжинэ)
  - *Санхүү* — нийт өртөг/орлого/ашиг, алдагдалтай бүс, ROI-гаар доод/дээд 6 бүс,
    мөн **худалдаалагдах орлогын задаргаа**. Бүс сонгоход задаргаа **тэр бүсээр**
    хязгаарлагдана; зориулалтын мөр дээр дарахад газрын зураг дээр **зөвхөн тэр
    зориулалтын барилга** үлдэж шүүгдэнэ (дахин дарж цуцална)
  - *Үзүүлэлтээр* — нормын муж диаграм дээр бүс бүрийн утгыг цэгээр байрлуулж,
    статистик болон доод/дээд 6 бүсийг харуулна
- **Багц бүс**: газрын зураг дээр том, шаравтар шошготой, эрэмбэд ◆ тэмдэгтэй.
- **Бүсийн дэлгэрэнгүй панель**: зургийн **зүүн дээд буланд** нээгдэнэ. Толгой
  хэсгээс нь чирж чөлөөтэй зөөнө (зургийн хүрээнээс гарахгүй). Өөр бүс сонгоход
  зөөсөн байрлалдаа үлдэнэ; хаагаад дахин нээхэд буланд буцна.
- **Багана өргөсгөх**: зүүн ба баруун талбарын хажуугийн зураасыг чирж өргөнийг
  өөрчилнө (220–620 px). Давхар товшиход 330 px анхны хэмжээнд буцна. Сонголт нь
  `localStorage`-д хадгалагдаж дараагийн удаа сэргэнэ.
- **Барилгын дүрслэл**: 368 барилга газрын зураг дээр (бүсийн будалтын дээр).
  4 горимоор өнгөлнө — *зориулалтаар* (орон сууц / үйлчилгээ / боловсрол /
  нийгмийн / инженер / зогсоол / бусад), *төлөвөөр* (төлөвлөсөн / баригдаж
  байгаа / одоо байгаа), *худалдаалагдах эсэхээр* (шүүлтүүрийг дагаж шинэчлэгдэнэ),
  *давхрын тоогоор*. Барилга дээр **хулгана аваачихад** hover панель гарч
  зориулалт, төлөв, ашигтай/нийт талбай, давхар, өрх, оршин суугч (эсвэл хүчин
  чадал), зогсоол, бүс, мөн санхүүд орох эсэхийг харуулна. Popup ашиглахгүй.

## FAR — эх өгөгдлийн эвдэрсэн талбар

Бүсийн `FAR` талбар **52 бүсийн 22-т эвдэрсэн**: утга таслагдаж 1.15-ын оронд
0.01, 8.43-ын оронд 0.08 гэж бичигджээ. `FAR_HUVI` (хувиар) нь бүрэн бүтэн
бөгөөд `BAR_M2 / GAZAR_M2`-тай 43/52 бүст таарна.

Тиймээс **`FAR_HUVI ÷ 100`-г засварласан утга болгон шууд ашиглана**
(анхааруулга биш — утгыг нь залруулж байгаа).

|---|---|---|---|
| B-2.1 | 0.01 | 115.11 | **1.15** |
| A-27 | 0.08 | 843.14 | **8.43** |
| D-11 | 0.03 | 324.98 | **3.25** |

## Хүн ам: оршин суугч ба хүчин чадал

`Total_population` талбар нь барилгын зориулалтаас хамаарч **өөр утга** заана:

| | Хүн | Өрх | Тайлбар |
|---|---|---|---|
| Орон сууц, town/single house | **43,287** | 12,381 | **оршин суугч** — 43,287 / 12,381 = яг **3.50 хүн/өрх** |
| Бусад (сургууль, үйлчилгээ…) | 25,039 | **0** | **хүчин чадал** — «Сургууль 960 хүүхэд» × 4 = 3,840 гэж яг таарна |
| Нийт | 68,326 | 12,381 | |

Тиймээс **хүн амын нягтшил** болон **нэг хүнд ногдох ногоон байгууламж**-ийг
зөвхөн **оршин суугчаар** бодно ([data.js](js/data.js) → `isResidential`).
Нийт дүнгээр бодвол нягтшил **58%-иар хөөрөгдөж**, орон сууцандаа тоологдсон
хүүхдийг сургууль дээр нь давхар тооцно.

Бүсийн дэлгэрэнгүйд «Оршин суугч» ба «Үйлчилгээний хүчин чадал» тусад нь харагдана.

## Талбайн гурван эх сурвалж

| Талбар | Хаана ашиглав | Тэмдэглэл |
|---|---|---|
| **`Area`** (га) | **Санхүү** — газрын хөрөнгө оруулалт | 52 бүсийн 49-д нь `Shape__Area`-тай тэнцүү |
| **`Shape__Area`** (м²) | **Хот төлөвлөлт** — нягтшил, FAR, BCR | Сервер талын бодит полигоны талбай |
| `GAZAR_M2` / `GAZAR_GA` | **ашиглахгүй** | Алдаатай бичлэгүүдтэй (ж: B-2.1 нь 3,680 м² гэж бичигдсэн ч бодит нь 24,366 м²) |

`Area` нь дараах 3 бүс дээр л полигоноос зөрнө — зам, нийтийн эзэмшил хассан
цэвэр талбай. **Эдгээр утга зөв болохыг өгөгдөл эзэмшигч баталсан.**
Бусад 49 бүст ялгаагүй:

|---|---|---|---|

| Багц-3.3 | 3.91 га | 6.96 га | 6.09 га |

Эдгээр бүсийг сонгоход санхүүгийн хэсэгт тайлбар тэмдэглэл гарна (алдаа биш).

## Өгөгдлийн чанарын анхаарах зүйл

- `A-17`, `D-8` бүсэд суурийн талбайн нийлбэр талбайгаасаа давсан хэвээр байна
  (BCR 130% / 113%) — барилга бүсийн зааг дээр давхцан хуваарилагдсан байж болзошгүй.
- Ийм тохиолдлыг бүсийн дэлгэрэнгүй хэсэгт **⚠ анхааруулга**-аар харуулна.

## БНбД босго утга — эх сурвалж

Үндсэн эх сурвалж: **[БНбД 30-01-24](https://mcis.gov.mn/storage/uploads/process/202403/file_1710112307903138_162364495749810.pdf)**

### Хүснэгт 6.1 — FAR / BCR норматив

| Бүсчлэл | BCR | FAR |
|---|---|---|
| Олон давхар олон айлын орон сууц (7–16 давхар) | 0.4 | 1.2 |
| Олон давхар, дахин төлөвлөлт (7–16) | 0.6 | 1.6 |
| Дундаж / нам давхар (3–6 давхар) | 0.4 | 0.8 |
| Блокчилсон (таунхаус, твинхаус) | 0.3 | 0.6 |
| 1–2 айлын орон сууц, гэр сууц | 0.2 | 0.4 |
| Олон төрлийн (нийгэм, олон нийтийн) | 1.0 | 3.0 |
| Нийгэм, олон нийтийн төрөлжсөн | 0.8 | 2.4 |

> **Хараахан хэрэгжүүлээгүй.** Одоогоор бүх бүсэд ганц муж (FAR 1.0–2.5,
> BCR 20–40%) хэрэглэж байна. Бүсийн `TOROL`-оор нь холбох шийдвэр хүлээгдэж байна.

### ⚠ Баримт бичгийн зөрчил

1. **Зүйл 6.8 өөртэйгөө зөрчилдөнө** — «Хүснэгт 6.1-д заасан хэмжээнээс **ихгүй**»
   гэсний дараа «...FAR, BCR-аар ... норматив үзүүлэлтээс **багагүй**» гэж бичсэн.
   ТӨСӨЛ хувилбарын засварын алдаа бололтой.
2. **FAR-ийн томьёо буруу** — баримт бичигт `м² ÷ га × 100` гэж бичсэн (зөв нь м²/м²).
   Апп зөв томьёогоор боддог.

### ⚠ Хараахан батлагдаагүй (таамаг)

- **Алхалтын бүсийн хамрах хүрээ 60%** — БНбД-д хувиар илэрхийлсэн норм алга
  (Хүснэгт 8.2 нь 15/20 минутын хүрээгээр заадаг)
- **Инженерийн шугам хүртэлх зай 100 м**
